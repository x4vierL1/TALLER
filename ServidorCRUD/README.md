1. ¿Qué es Node.js?
Node.js es un entorno que permite ejecutar código de JavaScript fuera del navegador,
 principalmente para crear servidores y aplicaciones web.

 2. ¿Qué es un servidor HTTP?
Es un programa que recibe solicitudes de los usuarios, responde enviando información, como páginas, imágenes o datos.

3. ¿Cuál es la diferencia entre Node.js y Express?
Node.js es el entorno donde se ejecuta JavaScript.
Express es un framework que funciona sobre Node.js y facilita la creación de servidores y aplicaciones web.

4. ¿Qué hace el módulo http?
El módulo http permite crear un servidor HTTP en Node.js para recibir solicitudes y enviar respuestas.

5. ¿Qué significa REST?
REST significa Representational State Transfer. 
Es un estilo para crear APIs que permite la comunicación entre aplicaciones usando HTTP.

6. ¿Qué es una API?
Una API es un conjunto de reglas que permite que dos aplicaciones se comuniquen e intercambien información.

7. ¿Qué diferencia existe entre GET y POST?
GET: Se utiliza para obtener o consultar información.
POST: Se utiliza para enviar información y crear nuevos datos.

8. ¿Qué diferencia existe entre PUT y PATCH?
PUT: Reemplaza toda la información de un recurso.
PATCH: Actualiza solo una parte de la información.

9. ¿Qué hace DELETE? 
DELETE se utiliza para eliminar un recurso o un dato del servidor.

10. ¿Qué es JSON?
 es un formato de texto que se utiliza para almacenar e intercambiar datos entre aplicaciones. Es fácil de leer y escribir.


Nombre del proyecto
Gestion de Clientes y Productos 

objetivo
El objetivo del proyecto es desarrollar un servidor backend que permita administrar
información de productos y clientes mediante operaciones CRUD (Crear, Leer, Actualizar y Eliminar).

Tecnologías utilizadas

- Node.js: Entorno de ejecución del servidor.
- Thunder Client: Herramienta utilizada para realizar pruebas de las rutas.
- Git/GitHub: Control de versiones del proyecto y almacenamiento del mismo
- claude, codex: apoyo sobre codigo para mejor control y reduccion de errores, 
  implementaciones importantes etc
